Le projet

Kammthaar est un projet d'application de gestion d'énergie solaire conçu pour les systèmes embarqués tels qu'un camion aménagé. Ce projet se concentre sur l'optimisation de l'utilisation de l'énergie solaire tout en garantissant une accessibilité aux données, même dans des environnements où la connectivité est limitée.

Objectif

Kammthaar vise à fournir une solution complète et efficace permettant : - La surveillance et l'analyse en temps réel des performances du système solaire. - L'enregistrement et la gestion des données pour un usage immédiat ou ultérieur. - Une utilisation flexible grâce à des scénarios de connectivité en ligne et hors ligne.

Fonctionnalités principales
  • Récupération des données en temps réel

    Les données, telles que la tension, le courant, la puissance des panneaux solaires, ainsi que l'état de charge des batteries, sont récupérées directement depuis le contrôleur MPPT via Modbus RTU, grâce au Raspberry Pi 5.

  • Gestion des données

    Les données sont principalement stockées dans une base de données InfluxDB distante. En cas d'absence de connexion réseau, elles sont temporairement sauvegardées dans un fichier JSON sur le Raspberry Pi puis synchronisées automatiquement dès qu'une connexion est rétablie.

  • Statistiques détaillées

    Les statistiques sont directement extraites du contrôleur MPPT, permettant une analyse précise des performances sans nécessiter de calculs complexes sur le backend.

  • Visualisation des données

    L'application Angular offre une interface utilisateur intuitive pour afficher les données en temps réel et consulter les statistiques historiques, garantissant une expérience utilisateur fluide et réactive.

  • Conception robuste

    - Le Raspberry Pi exécute un backend Flask qui agit comme un collecteur de données locales.
    - Un backend Flask sur un serveur distant centralise les communications entre Angular, InfluxDB et le backend Flask du Raspberry Pi si ce dernier est en ligne.
    - La base de données InfluxDB est utilisée pour gérer efficacement les séries temporelles, permettant une gestion optimale des données historiques et en temps réel.
    Cette approche garantit une séparation claire des responsabilités et une sécurité renforcée, car Angular ne communique jamais directement avec le Raspberry Pi.
